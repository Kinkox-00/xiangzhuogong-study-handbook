## 数值表示

必会内容：

- 补码范围、符号扩展、截断、溢出。
- 小端法：低字节存在低地址。
- IEEE 浮点：符号位、阶码、尾数、偏置、规格化、非规格化、特殊值。
- 向偶数舍入。
- 区分“数值转换”和“指针强转后的位级重新解释”。

浮点题步骤：

1. 写符号位。
2. 把绝对值转成二进制科学计数法。
3. 写真实阶码。
4. 加偏置得到阶码字段。
5. 写尾数字段。
6. 尾数不够时按题目舍入规则处理。

::: details 点击展开：小端法记忆卡

小端法 Little Endian：低字节存低地址，高字节存高地址。

例：

$$
0x12345678
$$

从低地址到高地址依次存为：

$$
78\quad 56\quad 34\quad 12
$$

易错点：不要把十六进制从左到右的书写顺序，误认为内存从低地址到高地址的存储顺序。

:::

## 汇编与 C 互推

32 位 x86 汇编互推是最高收益题型。读长汇编时不要逐行死读，先抓五类线索：

1. 初始化。
2. 标签和跳转。
3. 比较指令。
4. 变量更新。
5. 返回值。

AT&T 格式规则：

| 指令 | 记忆 |
|---|---|
| `movl 源, 目的` | 源在前，目的在后 |
| `leal` | 常用于地址计算或乘加表达式 |
| `cmpl a,b` | 比较的是 $b-a$ |
| `testl a,a` | 常用于判断 $a$ 是否为 0 或正负 |
| `call` | 会自动把返回地址压栈 |
| `xchg %ax,%ax` | 通常等价于 `nop`，可当作对齐填充 |

### 变量表方法

遇到汇编填空，先把偏移和变量对应起来：

| 位置形式 | 可能含义 |
|---|---|
| `8(%ebp)` | 第 1 个函数参数 |
| `12(%ebp)` | 第 2 个函数参数 |
| `-4(%ebp)` | 第 1 个局部变量或临时变量 |
| `0x1c(%esp)` | main 局部变量区中的某个槽位 |

易错点：`8(%ebp)` 是栈帧中的地址偏移，不代表变量大小。结构体对象可能是 12 字节，但结构体指针参数在 32 位机上仍是 4 字节地址。

::: details 点击展开：函数调用栈帧模板

函数调用题固定画三层：

1. 调用前：实参从右到左压栈。
2. `call`：CPU 自动压入返回地址。
3. 进入被调函数：`push %ebp; mov %esp,%ebp` 建立新栈帧。

进入被调函数后，常见布局是：

| 位置 | 含义 |
|---|---|
| `0(%ebp)` | old EBP |
| `4(%ebp)` | 返回地址 |
| `8(%ebp)` | 第 1 个参数 |
| `12(%ebp)` | 第 2 个参数 |

记忆卡片：参数不是看“压栈顺序谁先出现”，而是看进入函数后的标准栈帧位置。

:::

::: details 点击展开：结构体与指针题模板

先写字段偏移表。例如：

| 字段 | 偏移 |
|---|---:|
| `value` | 0 |
| `lchild` | 4 |
| `rchild` | 8 |

若 `ps` 是结构体指针，则：

$$
ps \to value
$$

等价于：

$$
(*ps).value
$$

易错点：`->` 已经包含“通过指针取字段”的含义，不要写成 `ps->*pnext`。

:::

## 虚拟内存与 Cache

地址翻译题固定流程：

1. 虚拟地址拆成 VPN 和 VPO。
2. 用 VPN 查 TLB。
3. TLB 未命中时查页表。
4. 页表 valid 为 1 时得到 PPN；valid 为 0 时缺页。
5. 物理地址由 PPN 和 PPO 组成。
6. 物理地址再进入 Cache，拆成 tag、set index、block offset。

页内偏移不变：

$$
VPO=PPO
$$

字段位数：

| 字段 | 由什么决定 |
|---|---|
| VPO/PPO | 页大小 |
| VPN | 虚拟地址总位数减 VPO |
| PPN | 物理地址总位数减 PPO |
| TLBI | TLB 组数 |
| TLBT | VPN 剩余位 |
| CO | Cache 块大小 |
| CI | Cache 组数 |
| CT | 物理地址剩余位 |

::: details 点击展开：TLB 未命中不等于缺页

TLB 是页表的缓存。TLB 未命中只表示“快表里没找到”，接下来还要查页表。

若页表项 valid 为 1，则仍然可以得到 PPN，地址翻译继续进行。

只有页表项 valid 为 0，才表示缺页，需要操作系统介入。

记忆主线：

$$
TLB\ miss \neq page\ fault
$$

:::

::: details 点击展开：组相联 Cache 的取模对象

组相联 Cache 里：

$$
组号=块号\bmod 组数
$$

替换只发生在这个组内部，再按 LRU、FIFO 或随机策略选择某一路。

易错点：2 路组相联、有 4 行时，不是对 4 取模，而是先分成 2 组，每组 2 路，所以对组数 2 取模。

:::

## 链接重定位与符号

链接题先问三件事：

1. 这个名字是否分配存储？
2. 别的文件能不能链接到它？
3. 它在 ELF 的哪个节里？

常见节：

| 节 | 内容 |
|---|---|
| `.text` | 机器指令 |
| `.rodata` | 字符串常量和只读数据 |
| `.data` | 已初始化全局变量和静态变量 |
| `.bss` | 未初始化或初始化为 0 的全局/静态变量 |
| `.symtab` | 符号表 |
| `.rel.text` / `.rel.data` | 重定位信息 |

强弱符号记忆：

| 对象 | 常见属性 |
|---|---|
| 函数定义 | 强符号 |
| 已初始化全局变量 | 强符号 |
| 未初始化全局变量 | 常按弱符号或 COMMON 处理 |
| 局部变量 | 不是链接器要解析的全局符号 |
| 宏 | 预处理阶段被替换，不是符号 |

::: details 点击展开：重定位题模板

链接器主要做三件事：

1. 符号解析：把引用对应到某个定义。
2. 节合并与地址分配：给 `.text/.data/.bss` 等分配最终地址。
3. 重定位：修改代码或数据中原本未知的地址或位移。

看到 `.text` 中的外部函数调用：

$$
call\ rel32
$$

常见重定位类型是 `R_386_PC32`。其核心计算是：

$$
rel32=目标地址-下一条指令地址
$$

若是把符号绝对地址装入寄存器或内存，常见为 `R_386_32`。

:::

## 进程信号与程序优化

进程与信号题常见关键词：

- `fork`
- `execve`
- `wait`
- `waitpid`
- `SIGCHLD`
- signal handler

`SIGCHLD` 题的稳定答法：

1. 子进程退出时，内核向父进程发送 `SIGCHLD`。
2. 标准信号可能合并，多个子进程退出不一定收到多次信号。
3. 若 handler 只调用一次 `wait`，可能只回收一个子进程。
4. 结果可能留下僵尸进程。
5. 改法是在 handler 中用 `waitpid(-1,&status,WNOHANG)` 循环回收。

程序优化题常见答法：

- 循环展开减少循环控制开销。
- 多累加器打破单一累加变量的数据相关。
- 提高指令级并行。
- 局限：代码膨胀、寄存器压力、整数溢出、小规模输入收益有限。

::: details 点击展开：循环展开与多累加器模板

加法求和可以写成：

```c
int sum0 = 0, sum1 = 0, sum2 = 0, sum3 = 0;
int i = 0;
for (; i + 3 < n; i += 4) {
  sum0 += a[i];
  sum1 += a[i + 1];
  sum2 += a[i + 2];
  sum3 += a[i + 3];
}
int sum = sum0 + sum1 + sum2 + sum3;
for (; i < n; i++) sum += a[i];
```

卷面解释：减少循环控制开销，打破单一累加变量的数据相关，提高指令级并行。

:::
